# Moltrix Secure Endpoint Management

A hands-on Microsoft Intune and Microsoft Entra portfolio project aligned with MD-102 skills and enterprise endpoint-management practices.

## Project Status

- **Current phase:** Requirements documented; Phase 0 foundation review in progress
- **Environment:** Portfolio lab
- **Platform:** Microsoft Intune and Microsoft Entra
- **Certification alignment:** MD-102
- **Project charter:** Completed

> This is a controlled learning environment. All public evidence will be sanitized to protect identities, device information, tenant details, and credentials.

## Roadmap Context

This repository is the Phase 1 practical project within the [Moltrix Cloud & AI Transformation Program](https://github.com/AnwarAliCloudAI/moltrix-cloud-ai-roadmap).

Phase 0 develops the operating-system, networking, DNS, identity, virtualization, automation, and troubleshooting foundations required to make sound endpoint-management decisions.

Phase 1 applies those foundations to a real business challenge: securing and managing Moltrix users, devices, applications, and organizational information.

The architecture decision will be completed after the relevant Phase 0 identity and networking foundations have been reviewed.

## Business Scenario

Moltrix requires a secure and manageable endpoint solution for a small distributed workforce. Users need controlled access to organizational applications and information from acceptable devices.

The project will examine how Microsoft Entra ID and Microsoft Intune can support identity preparation, device enrollment, configuration management, compliance evaluation, application deployment, endpoint security, testing, and troubleshooting.

## Central Project Question

> How can Moltrix allow employees to work productively from different locations and devices while centrally managing security, access, applications, and organizational information?

## Version 1 Objectives

- Document the business, technical, and security requirements.
- Compare traditional, cloud-native, and hybrid identity and management approaches.
- Select and explain the Moltrix endpoint-management architecture.
- Prepare Microsoft Entra test users and groups.
- Enroll and validate a test Windows device.
- Apply configuration and compliance policies.
- Deploy and validate one application.
- Apply selected endpoint-security controls.
- Test successful and unsuccessful scenarios.
- Document troubleshooting decisions and lessons learned.

- Identity concepts will continue into Microsoft 365 and Azure.
- Device compliance will support Zero Trust and governance decisions.
- Networking and DNS will support Azure and AWS administration.

MD-102 is therefore one connected stage in the Moltrix journey, not an isolated certification.

## Security and Privacy Rules

The following information will not be published:

- Passwords, secrets, API keys, tokens, or recovery codes
- Tenant or subscription identifiers
- Personal user information
- Device serial numbers or hardware identifiers
- Unredacted administrative screenshots
- Confidential organizational information

Evidence sanitization is part of the project acceptance standard.
